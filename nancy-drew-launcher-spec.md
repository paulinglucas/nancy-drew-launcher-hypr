# Nancy Drew Launcher — Project Spec

## Overview

A native GTK4 game launcher for running Nancy Drew PC games (HerInteractive) under Wine on Arch Linux / Hyprland. The user has 20 Nancy Drew game directories (copied from a Windows install), each containing a `Game.exe`, `CDSound/`, `CDVideo/`, and `Ciftree/` folders, plus assorted data files. The launcher provides a noir-themed native window for browsing and launching these games, plays the Nancy Drew theme music while browsing, and uses Gamescope to handle display scaling properly.

## Goals

- One-click desktop icon → opens native launcher window on Hyprland
- Browse all 20 games in a stylish dark mystery/noir UI
- Nancy Drew theme music plays while browsing
- Click a game → Gamescope + Wine launches it at proper resolution
- Per-game configuration (resolution, winetricks overrides, etc.)
- Setup script that creates the Wine prefix and installs common dependencies
- Clean GitHub repo with a README that any Arch + Hyprland user could follow

## Tech Stack

- **GUI**: Python 3 + GTK4 (via PyGObject / `gi` bindings)
- **Audio**: GStreamer (via `gi.repository.Gst`) — comes with GTK
- **Game runner**: Gamescope + Wine
- **No web server, no browser, no Node.js, no Electron**

### Game Launch Logic (Gamescope + Wine)

These old Nancy Drew games expect 800x600 or 1024x768 and don't handle modern displays. Gamescope wraps them in a nested Wayland compositor that handles the resolution mismatch.

```python
def launch_game(self, game):
    config = self.config
    env = os.environ.copy()
    env["WINEPREFIX"] = os.path.expanduser(config["wineprefix"])
    
    games_dir = os.path.expanduser(config["games_dir"])
    exe_path = os.path.join(games_dir, game["folder"], game["exe"])
    work_dir = os.path.dirname(exe_path)
    
    settings = game.get("settings", config["default_settings"])
    
    for k, v in settings.get("env_vars", {}).items():
        env[k] = v
    
    cmd = []
    
    if settings.get("use_gamescope", True):
        gw = settings.get("game_width", 1024)
        gh = settings.get("game_height", 768)
        ow = settings.get("output_width")
        oh = settings.get("output_height")
        
        cmd += ["gamescope", "-w", str(gw), "-h", str(gh)]
        if ow and oh:
            cmd += ["-W", str(ow), "-H", str(oh)]
        cmd += ["-f", "--"]
    
    cmd += ["wine", exe_path]
    
    subprocess.Popen(cmd, env=env, cwd=work_dir,
                     stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL)
```

**Why Gamescope:**
- Solves the black screen problem (game tries to set display mode and fails on Wayland)
- Solves the tiny window problem (game renders at 1024x768, Gamescope scales it up)
- Works natively on Hyprland (both are Wayland)
- No need for `wine explorer /desktop=` hack

**Fallback:** if `use_gamescope` is `false`, launch with `wine explorer /desktop=NancyDrew,1024x768 Game.exe` instead.

### `setup.sh`

Interactive first-time setup script. Responsibilities:

1. Check/prompt for multilib in pacman.conf
2. Install all Arch dependencies via pacman
3. Ask for games directory path (default: ~/Games/NancyDrew)
4. Create 32-bit Wine prefix: `WINEPREFIX=<path> WINEARCH=win32 wineboot`
5. Install common winetricks deps: `d3dx9 quartz wmp9 vcrun6`
6. Optionally install `quicktime72` for older titles (games 1-7)
7. Download and install launcher fonts to `~/.local/share/fonts/`
8. Scan games directory, match folders to known game metadata, generate `config.json`
9. Install `.desktop` file to `~/.local/share/applications/` with absolute paths
10. Print summary

Must be idempotent — safe to run multiple times.

## Notes for Claude Code

- Use `python-gobject` and `gtk4` from Arch repos — not pip
- `libadwaita` (Adw) is optional but gives AdwApplication which handles dark theme natively
- GTK4 CSS is NOT web CSS — it supports a subset. Key differences:
  - No flexbox/grid layout — use GtkBox, GtkFlowBox, GtkGrid widgets
  - Use `@define-color` instead of CSS variables
  - Transitions are limited: `transition: <property> <duration> <easing>`
  - Pseudo-classes: `:hover`, `:active`, `:focus`, `:selected`, `:checked`
  - Use `background-color` not `background` shorthand for solid colors
- For the game card grid, use `Gtk.FlowBox` with `set_homogeneous(True)` and `set_max_children_per_line(4)`
- Each card is a `Gtk.Box` inside a `Gtk.FlowBoxChild` with CSS classes applied
- Search filtering: use `Gtk.FlowBox.set_filter_func()`
- GStreamer: `playbin` element with `uri` property is the simplest approach for music
- All paths in config.json use `~` — expand with `os.path.expanduser()` at runtime
- Handle missing `theme.mp3` gracefully (just skip music, no crash)
- `subprocess.Popen` for launching games — NOT `subprocess.run` (that blocks the UI)
- Redirect game stdout/stderr to `/dev/null`
- The `.desktop` file must use absolute paths — `setup.sh` templates them
- `setup.sh` must be idempotent
- Gamescope is in the official Arch repos
- GTK4 windows float by default on Hyprland (not tiled) — this is correct behavior
